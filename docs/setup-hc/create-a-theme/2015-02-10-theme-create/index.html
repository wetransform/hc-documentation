Create theme

A theme is a re-usable configuration object used to define common settings for data set metadata, service publishing and transformation. The theme manager manages themes and the associated schemas and transformation projects.

The process for the creation of a theme is typically:

  1. Create Schema
  2. Upload transformation project
  3. Create theme
  4. Edit theme

Once you have created the schema and transformation project, follow these steps as a logged in theme manager or organisation superuser to create a new theme:

  1. Go to «Themes»
  2. Go to «Create new theme» at the bottom of the theme list
  3. Select base theme. In the first step, decide whether you want to re-use an existing theme configuration by deriving your new theme from the selected one. You will be able to edit the configuration later on. Click «Continue»
  4. Select name and type. In the second step, select your theme type and assign a name, version and description to your theme. You can choose from one of the existing, preconfigured theme types in the dropdown menu, or choose Custom to create your own theme type. Click «Continue»
  5. Add feature types. In the third step, click «Add new type» to add feature types to your theme. Click on the » button to select a schema that contains the feature types you would like to add. Next, click the » button to select a single feature type or click «Add all» to add all of the listed feature types to your theme. When you are finished, click «Continue»

You can assign as many features types as you want to a theme. A single feature type can be associated with many themes. If you want to remove the assignment of a single feature type, click the red «remove» icon in the top right corner of the feature type. If you want to remove all feature types of a single schema currently associated to the theme, click the red «remove» icon in the top right corner of the schema. When you are finished adding feature types, click «Continue» 6. Confirm creation. In the fourth step, review your theme and click «Create» to save the theme.

info

When you derive your theme from an existing one, later changes to the original theme do not influence your derived theme.

caution

You can only delete a theme when no data sets are associated with it anymore.
